Специальный модуль test_kernel содержит набор тестов предназначенных для тестирования различных подсистем и узлов системы OpenSCADA. Тесты запускаются с указанным периодом. Конфигурация тестов производится через конфигурационный файл системы OpenSCADA.\\

Конфигурационные поля тестов помещаются в секцию модуля test_kernel. Формат конфигурационных полей:\\
\textbf{<prm id="Test Id" on="1" per="10" />}\\
где:
\begin{itemize}
\item id - идентификатор теста;
\item on - признак "Тест включен";
\item per - период повторения теста (секунд);
\end{itemize} 
Кроме основных атрибутов каждый тест может иметь несколько собственных атрибутов.\\

Версия 1.0.0 содержит следующий набор тестов: XML, MESS, PARAM, SOAttDet, Val, BD, TrOut.\\

\section{XML}
Тест XML-парсера. Парсит и отображает структуру указанного файл.\\
\textit{<prm id="XML" on="1" per="5" file="./3FC4B8E7.xml"/>}\\
Атрибуты:
\begin{itemize}
\item file - имя входного файла;
\end{itemize}

\section{MESS}
Тест архива сообщений. Периодически вычитывает новые сообщения из указанного архива.\\
\textit{<prm id="MESS" on="1" per="10" arh="sys_arh" t_arh="base_arh" categ=""/>}\\
Атрибуты:
\begin{itemize}
\item arh - имя тестируемого архива;
\item t_arh - тип тестируемого архива;
\item categ - категория запрашиваемых сообщений;
\end{itemize}

\section{PARAM}
Тест параметра. Вычитывает конфигурационные поля, атрибуты параметра и их значения. Парсит и отображает узел языка управления для параметра.\\
\textit{<prm id="PARAM" on="1" per="15" name="TEST_VirtualC"/>}\\
Атрибуты:
\begin{itemize}
\item name - имя параметра;
\end{itemize}

\section{SOAttDet}
Тест на подключение/отключение модуля. С указанным периодом выполняется  подключение и отключение указанного модуля.\\
\textit{<prm id="SOAttDet" on="1" per="30" name="/usr/lib/openscada/ui_web_dbg.so" full="1"/>}\\
Атрибуты:
\begin{itemize}
\item name - имя модуля;
\item full - признак полного, с запуском, подключения;
\end{itemize}

\section{Val}
Тест атрибута параметра. Выполняет периодический опрос значения указанного атрибута.\\
\textit{<prm id="Val" on="1" per="1" name="T_CPU/value"/>}\\
Атрибуты:
\begin{itemize}
\item name - имя атрибута в виде [параметр]/[атрибут];
\end{itemize}

\section{BD}
Полный тест БД. Выполняет полную процедуру:
\begin{itemize}
\item создание/открытие БД;
\item создание/открытие таблицы;
\item создание множества полей(строк) предопределённой структуры;
\item модификация множества полей;
\item получение и проверка значений множества полей;
\item проверка списка полей;
\item удаление полей;
\item закрытие/удаление таблицы;
\item закрытие/удаление БД;
\end{itemize}
\textit{<prm id="BD" on="1" per="10" type="direct_dbf" bd="./DATA1" table="test.dbf" size="1000"/>}\\
Атрибуты:
\begin{itemize}
\item type - тип БД;
\item bd - имя БД;
\item table - таблица;
\item size - количество создаваемых полей в таблице;
\end{itemize}

\section{TrOut}
Тест исходящего транспорта. Выполняет тестирование исходящего транспорта путём отправления запроса к указанному входящему транспорту.\\
\textit{<prm id="TrOut" on="0" per="10" addr="UDP:127.0.0.1:daytime" type="socket" req="time"/>}\\
Атрибуты:
\begin{itemize}
\item addr - адрес входящего транспорта;
\item type - тип исходящего транспорта;
\item req - запрос;
\end{itemize}
